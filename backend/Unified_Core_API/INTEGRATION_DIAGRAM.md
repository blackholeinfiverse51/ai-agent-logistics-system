## Event Flow Types

### 1. Synchronous Flow (Request-Response)
```
Client â†’ API Endpoint â†’ Module Logic â†’ Response â†’ Client
```
**Example**: `POST /crm/leads` returns lead ID immediately

### 2. Asynchronous Flow (Event-Driven)
```
Client â†’ API Endpoint â†’ Event Published â†’ Event Broker â†’ Subscribers â†’ Processing
```
**Example**: Order creation triggers multiple downstream actions

### 3. Hybrid Flow (Sync + Async)
```
Client â†’ API Endpoint â†’ Immediate Response + Event Published â†’ Async Processing
```
**Example**: `POST /event/unified` returns success immediately, processing happens async

## Scalability Strategy

### Horizontal Scaling
- API instances: 3-10 replicas
- Load balancing: Round-robin / Least connections
- Session management: Redis-based

### Vertical Scaling
- CPU: 2-4 cores per instance
- Memory: 2-4 GB per instance
- Database: Read replicas for queries

### Caching Strategy
- Redis for session data
- API response caching (5 min TTL)
- Database query caching
